## Common Pitfalls: Why Rewards Don't Pay Out

A surprising number of "OKX referral bonus is a scam" complaints trace back to a small set of avoidable mistakes. Here are the ones I see repeatedly:

- **Referral code entered after account creation** — Once the account exists, the code can't be added retroactively. Always enter the code on the registration form, before completing sign-up.
- **Tasks not completed within 14 days** — The deposit and trading task window is short. If you sign up and then disappear for three weeks, you've forfeited the welcome rewards.
- **Wrong deposit method** — Internal transfers, red packets, rewards, and P2P shared orders don't count toward deposit tasks. Use on-chain transfers or fiat deposits.
- **Cumulative trades instead of a single qualifying trade** — The Full Rebate Program requires one trade that meets the minimum threshold. Ten small trades that add up to the threshold don't qualify.
- **Zero-fee pairs used for the First Qualifying Trade** — USDC/USDT, USDG/USDT, USD1/USDT, USDT/USDS, PYUSD/USDT, and Convert transactions are all excluded.
- **Forgot to enroll before trading** — If you place your First Qualifying Trade without first tapping "Enroll" on the program landing page, you may not be eligible.
- **Forgot to claim daily rewards** — Each day's Full Rebate installment must be manually claimed within its validity window. Miss it and it's gone forever.
- **Cross-region referral** — If you invite someone outside your region (e.g., an APAC user inviting an EEA user), the inviter reward may not be issued.
- **Used perks to hit trading milestones** — Trades funded by rebate cards, contract vouchers, or welcome bonuses typically don't count as valid trades for campaign tasks.
- **Risk-control flags** — Accounts flagged as high-risk lose eligibility for both enrollment and already-distributed rewards.

None of these are hidden — they're all in OKX's published terms — but they're easy to miss in the excitement of opening a new account.
